# Contexto y Reglas del Proyecto: Web de Servicios de PC

## 📌 Contexto del Proyecto
Este proyecto consiste en el diseño y desarrollo de una página web profesional, dinámica y moderna para promocionar servicios técnicos de computadores (PC). El propietario del sitio cuenta con conocimientos en programación, mantenimiento de hardware, configuración de equipos, armado de PC custom (gaming/oficina), formateo, instalación de sistemas operativos y activación de paquetería de software (Microsoft Office, etc.).

## 🎯 Objetivo Principal
* **Generar Ingresos**: El propósito fundamental del sitio es ganar dinero atrayendo clientes locales y facilitando que soliciten servicios técnicos a domicilio de manera rápida y directa.
* **Atacar la Necesidad del Cliente**: Mostrar de forma llamativa al usuario por qué necesita un técnico profesional (ej. resolver lentitud, eliminar virus, optimizar componentes, evitar fallas graves).
* **Conversión Directa**: Redirigir a los clientes directamente a un enlace de WhatsApp preconfigurado con un mensaje de solicitud de servicio para cerrar la venta inmediatamente.

## 🛠️ Tecnologías y Estructura
* **Tecnología**: React + Vite para un rendimiento óptimo y carga instantánea.
* **Estilos**: CSS Vanilla moderno con variables personalizadas.
* **Diseño Visual**: Estilo premium "Tech/Gamer". Fondo oscuro, tonos y acentos azules que inspiran tecnología, efectos de cristal (glassmorphic panels) y micro-animaciones interactivas al pasar el cursor.
* **Calidad Web**: SEO técnico estructurado (JSON-LD LocalBusiness) y accesibilidad (A11y) con soporte para navegación por teclado.

## ⚙️ Reglas de Desarrollo (Para el Asistente de IA)
1. **Idioma**: Todos los textos de la interfaz, comentarios del código y respuestas deben redactarse obligatoriamente en **español**.
2. **Control de Git**: NO realizar commits ni confirmaciones bajo ninguna circunstancia sin permiso explícito del usuario.
3. **Alineación Visual**: Conservar la coherencia estética de tonos oscuros y azules, asegurando un diseño totalmente responsivo (móvil y escritorio).
4. **Optimización**: Cada nuevo componente debe seguir las mejores prácticas de React (skills de `react-best-practices` y `seo` configurados en el proyecto).
